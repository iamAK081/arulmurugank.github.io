📋 Introduction

Managing a fuel station goes beyond selling petrol and diesel — operators must track sales, monitor inventory, handle credit transactions, and ensure ironclad accountability through audit logs. Fuel Station Management Audit Software, developed by Arulmurugan K, brings a modern, transparent, and role-based solution for Windows. Built with C# (WPF) and SQL Server, it provides real‑time visibility and a tamper‑evident audit trail.

🎯 Perfect For:

  • Independent fuel station owners & dealers
  • Fleet management & depot control
  • Multi‑pump automation with credit tracking
  • Audit‑ready fuel retail operations

⚙️ Technology Stack

Frontend: WPF / XAML (MVVM) Backend: C# / .NET 6 Database: SQL Server ORM: Entity Framework 6 Audit: JSON snapshots

🏗️ Application Architecture

Presentation Layer (WPF Views / MVVM) → Business Logic Layer (Services: Sales, Inventory, Credit) → Data Access Layer (EF + Repositories) → SQL Server + Audit Trail

✨ Core Features

  • 🔐 User Roles & Authentication: Operator, Admin, Super Admin (hashed passwords)
  • 💰 Sales & Transactions: Fuel, lubes, multiple payment modes: cash, card, UPI, credit with due dates
  • 📦 Inventory Management: Real‑time stock overview, low‑stock alerts, live updates
  • 📊 Reports Generation: Daily, monthly, yearly — CSV / JSON export. Stock / cash / credit statements
  • 📝 Audit Logs (JSON): Before/after snapshots on every insert/update/delete, user ID & timestamp
  • 🧾 Credit Tracking: Monitor pending collections, ageing of dues
  • 📎 Export & Backup: Automated report archiving, database backup hints
  • 🛡️ Role‑based UI: Different dashboards for operators and admins

🚀 Quick Start (C# snippet)

// Record a sale with audit (simplified service call)
public void RecordSale(SaleDto sale)
{
  using (var context = new FuelDbContext())
  {
  var beforeSnapshot = JsonConvert.SerializeObject(sale);
  var saleEntity = mapper.Map<Sale>(sale);
  context.Sales.Add(saleEntity);
  context.SaveChanges();
 
  // Audit after insert
  var audit = new AuditLog
  {
  Action = "INSERT", Table = "Sales", RecordId = saleEntity.Id,
  After = JsonConvert.SerializeObject(saleEntity),
  UserId = currentUserId, Timestamp = DateTime.UtcNow
  };
  context.AuditLogs.Add(audit);
  context.SaveChanges();
  }
}

❓ Common Issues

IssueSuggested fix
Login fails even with correct passwordVerify that the user is active and role assigned; check password hashing (BCrypt).
Inventory not updating after saleEnsure transaction scope includes both sale and stock deduction; check EF commit.
Audit logs missing JSON detailsVerify that Before/After snapshots are generated in the service layer; serializer settings.
