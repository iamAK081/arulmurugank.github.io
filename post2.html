📋 Overview – FuelStation Management (SQLite edition)

FuelManagement is a full Windows desktop application built with C# (WPF) and **SQLite**, designed for fuel station operators who need rigorous audit trails, credit sales tracking, and real‑time inventory. The repository (iamAK081/FuelManagement) uses SQLite for a zero‑configuration, single‑file database that simplifies deployment and backup.

🗄️ Why SQLite ?

The project originally used SQL Server but was optimized to SQLite for lightweight deployment – no separate database installation, ideal for small/medium fuel stations. All transaction data, inventory, user roles and audit logs are stored in a portable .db file. Entity Framework Core (or Microsoft.Data.Sqlite) handles data access seamlessly.

  • 📈 Credit reports: Detailed received payment report (Apr 2025)
  • 📝 Audit logs (JSON): Before/after snapshots per change, stored in SQLite
  • 📎 Export statements: CSV / JSON for daily, monthly
  • 🧾 Customer due ageing: Track pending collections via SQLite queries

🔧 SQLite integration example

// AppDbContext.cs – SQLite configuration (EF Core) public class AppDbContext : DbContext { public DbSet<Sale> Sales { get; set; } public DbSet<CreditPayment> CreditPayments { get; set; } public DbSet<AuditLog> AuditLogs { get; set; } protected override void OnConfiguring(DbContextOptionsBuilder options) => options.UseSqlite("Data Source=FuelStation.db"); } // Insert a credit payment (with audit) var payment = new CreditPayment { SaleId = 104, Amount = 1500, ReceivedDate = DateTime.Now }; context.CreditPayments.Add(payment); context.SaveChanges(); // single-file SQLite transaction
